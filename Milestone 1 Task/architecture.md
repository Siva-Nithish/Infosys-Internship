1. Where do logs come from?
Logs are generated by different system components such as:
•	Application servers
•	APIs
•	Databases
•	Background services

2. How are logs collected?
Logs are collected using a log collection mechanism (agent/collector).
The collector:
•	Continuously listens for new logs
•	Aggregates logs from multiple sources
•	Forwards logs to the processing pipeline

3. How are logs processed?
	Validation – Check structure and required fields
  Parsing – Convert raw logs into structured format
	Filtering – Remove unnecessary entries
	Analysis – Detect anomalies or rule violations

4. What happens to bad or broken logs?
They may be:
•	Sent to an error queue
•	Stored in a dead-letter store
•	Logged for debugging and auditing
